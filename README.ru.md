# uuu-gui

> [English version](README.md)

Кроссплатформенный графический интерфейс для [NXP UUU](https://github.com/nxp-imx/mfgtools) (Universal Update Utility) — стандартного инструмента прошивки процессоров NXP i.MX по USB.

![Platforms](https://img.shields.io/badge/platform-Linux%20%7C%20macOS%20%7C%20Windows-blue)

## Что это такое?

Процессоры NXP i.MX (i.MX 8, i.MX 8M, i.MX 6 и др.) используются во встраиваемых Linux-платах, промышленных компьютерах и кастомном железе. Когда плата находится в режиме восстановления (SDP — Serial Download Protocol), она подключается по USB и может быть прошита с помощью утилиты `uuu` от NXP.

`uuu` — мощный инструмент, но требует знания правильных аргументов командной строки, управления многофазными последовательностями прошивки и ручного отслеживания USB-пути каждого устройства. При прошивке нескольких плат одновременно — например, в производстве — это становится трудоёмким и чревато ошибками.

**uuu-gui** — десктопное Qt-приложение, построенное напрямую на `libuuu` от NXP, которое:

- Автоматически определяет NXP-устройства при подключении
- Позволяет создавать переиспользуемые пресеты прошивки (какие файлы и в каком порядке)
- Прошивает несколько устройств одновременно, каждое в отдельном процессе-хелпере
- Поддерживает многофазную прошивку: например, сначала загружает бинарник инициализации 4 ГБ RAM, затем прошивает полный образ eMMC
- Переживает смену USB-адреса устройства между фазами (после SDP-загрузки устройство появляется на новом порту)
- Показывает прогресс, метку текущей фазы и живой лог прошивки для каждого устройства
- Опционально записывает лог-файлы с временными метками для каждой сессии прошивки

Приложение **встраивает** `libuuu` от NXP (собирается из исходников) и работает с ним через маленький встроенный воркер `uuu-helper` — отдельный бинарник `uuu` устанавливать не нужно. Каждое устройство прошивается в своём процессе-хелпере, поэтому несколько плат можно шить параллельно с независимыми пресетами.

## Возможности

- **Пресеты прошивки** — сохранение и повторное использование конфигураций (файлы, тип, задержка между фазами)
- **Автоматическое определение устройств** — опрос `uuu-helper list` (libuuu) каждые 500 мс, без ручного ввода USB-пути
- **Параллельная прошивка** — каждое устройство получает отдельный процесс `uuu-helper`, отфильтрованный по серийному номеру
- **Многофазная прошивка** — для плат, требующих инициализации 4 ГБ RAM перед основной прошивкой eMMC
- **Обработка ре-энумерации USB** — после SDP-загрузки libuuu сам находит плату по серийному номеру на новом USB-адресе
- **Авто-прошивка при подключении** — автоматический старт прошивки при подключении устройства
- **Перезагрузка после прошивки** — отправка `FB: reboot` по завершении
- **Права по требованию** — прошивка идёт без прав; если доступ к устройству запрещён, приложение один раз за сессию спрашивает пароль и перезапускает прошивку через `sudo` (без выбора способа повышения прав)
- **Лог-файлы** — опциональная запись `.log`-файла с временной меткой для каждого устройства в UTF-8
- **Двуязычный интерфейс** — русский и английский, переключение без перезапуска

## Типы пресетов

| Тип | Описание |
|-----|----------|
| **SimpleBin** | Одна фаза: `uuu <file.bin>` — загрузка или простая SDP-прошивка |
| **EmmcAll** | Одна фаза: `uuu -b emmc_all <bootloader> <image.wic>` — полная прошивка eMMC |
| **EmmcAll4G** | Две фазы: `uuu <4g-init.bin>`, затем `uuu -b emmc_all <bootloader> <image.wic>` — для плат с 4 ГБ LPDDR4, требующих специального бинарника инициализации RAM перед основной прошивкой |

Пресеты сохраняются в настройках приложения и не теряются между сессиями.

## Требования

- **Qt 6.4+**
- **Зависимости сборки libuuu** — `libusb 1.0`, `libzstd`, `tinyxml2`, `bzip2`, `zlib`, `openssl` (libuuu скачивается и собирается из исходников на этапе конфигурации)
- Доступ в сеть при первой сборке (CMake `FetchContent` клонирует [nxp-imx/mfgtools](https://github.com/nxp-imx/mfgtools) на теге `uuu_1.5.243`)

Отдельный бинарник `uuu` не нужен — движок вкомпилирован.

## Сборка

### Linux

```bash
sudo apt install qt6-base-dev libqt6svg6-dev libusb-1.0-0-dev \
  libzstd-dev libtinyxml2-dev libbz2-dev zlib1g-dev libssl-dev \
  cmake build-essential git

cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build

./build/uuuapp
```

Для работы с USB без sudo добавьте udev-правило для устройств NXP:

```bash
echo 'SUBSYSTEM=="usb", ATTRS{idVendor}=="1fc9", MODE="0666"' | sudo tee /etc/udev/rules.d/70-nxp.rules
sudo udevadm control --reload-rules
```

Без этого правила приложение автоматически запросит пароль при ошибке прав доступа и перезапустит прошивку через `sudo`.

### macOS

```bash
brew install qt libusb zstd tinyxml2 bzip2 openssl@3 pkg-config

cmake -B build -DCMAKE_BUILD_TYPE=Release -DCMAKE_PREFIX_PATH="$(brew --prefix qt)"
cmake --build build

open build/uuuapp.app
```

Скрипт сборки сам определяет Homebrew и подставляет keg-only пути `pkg-config`
(zstd, tinyxml2) и `OPENSSL_ROOT_DIR`, так что нужен только путь до Qt.

При первом запуске macOS может заблокировать приложение, так как оно не подписано. Чтобы обойти это:

```bash
xattr -cr /path/to/uuuapp.app
```

Или: правой кнопкой → Открыть → Открыть в диалоге.

На macOS для захвата USB-устройств NXP через libusb требуется sudo — приложение само запрашивает пароль при ошибке доступа и перезапускает прошивку с повышенными правами.

### Windows

Зависимости через [vcpkg](https://github.com/microsoft/vcpkg):

```powershell
vcpkg install libusb zstd tinyxml2 bzip2 zlib openssl --triplet x64-windows
```

Установите [Qt 6](https://www.qt.io/download-qt-installer), затем:

```powershell
cmake -B build -DCMAKE_BUILD_TYPE=Release `
  -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build --config Release
windeployqt build/Release/uuuapp.exe
```

## Использование

1. Создайте **пресет прошивки** (левая панель → **Add**): выберите тип и файл(ы) для прошивки.
2. Подключите NXP-плату в режиме восстановления / SDP — она автоматически появится в списке устройств.
3. Выберите пресет и нажмите **Flash** в строке устройства, либо **Flash Checked Devices** для нескольких сразу.
4. Если доступ к устройству требует повышенных прав, приложение один раз спросит пароль и перезапустит прошивку через `sudo` (пароль кэшируется до конца сессии).

### Настройки

После запуска откройте **Settings**:

- **Language** — English / Русский (применяется сразу, без перезапуска)
- **Flash Logs** — включите для сохранения `.log`-файла с временной меткой для каждой сессии прошивки в выбранную папку.

Настраивать путь до `uuu` или способ повышения прав не нужно — движок `libuuu`
встроен, а права запрашиваются по необходимости.
