# uuu-gui

> [English version](README.md)

Кроссплатформенный графический интерфейс для [NXP UUU](https://github.com/nxp-imx/mfgtools) (Universal Update Utility) — стандартного инструмента прошивки процессоров NXP i.MX по USB.

![Platforms](https://img.shields.io/badge/platform-Linux%20%7C%20macOS%20%7C%20Windows-blue)

## Что это такое?

Процессоры NXP i.MX (i.MX 8, i.MX 8M, i.MX 6 и др.) используются во встраиваемых Linux-платах, промышленных компьютерах и кастомном железе. Когда плата находится в режиме восстановления (SDP — Serial Download Protocol), она подключается по USB и может быть прошита с помощью утилиты `uuu` от NXP.

`uuu` — мощный инструмент, но требует знания правильных аргументов командной строки, управления многофазными последовательностями прошивки и ручного отслеживания USB-пути каждого устройства. При прошивке нескольких плат одновременно — например, в производстве — это становится трудоёмким и чревато ошибками.

**uuu-gui** оборачивает `uuu` в десктопное Qt-приложение, которое:

- Автоматически определяет NXP-устройства при подключении
- Позволяет создавать переиспользуемые пресеты прошивки (какие файлы и в каком порядке)
- Прошивает несколько устройств одновременно, каждое в отдельном процессе `uuu`
- Поддерживает многофазную прошивку: например, сначала загружает бинарник инициализации 4 ГБ RAM, затем прошивает полный образ eMMC
- Автоматически определяет смену USB-адреса устройства между фазами (после SDP-загрузки устройство появляется на новом порту)
- Показывает прогресс, метку текущей фазы и живой вывод uuu для каждого устройства
- Опционально записывает лог-файлы с временными метками для каждой сессии прошивки

Приложение **не заменяет** `uuu` — для работы нужен отдельно скачанный бинарник `uuu`, который вызывается под капотом.

## Возможности

- **Пресеты прошивки** — сохранение и повторное использование конфигураций (файлы, тип, задержка между фазами)
- **Автоматическое определение устройств** — опрос `uuu -lsusb` каждые 500 мс, без ручного ввода USB-пути
- **Параллельная прошивка** — каждое устройство получает отдельный процесс `uuu` с флагом `-m busId`
- **Многофазная прошивка** — для плат, требующих инициализации 4 ГБ RAM перед основной прошивкой eMMC
- **Обработка ре-энумерации USB** — после SDP-загрузки устройство меняет USB-адрес; приложение находит новый адрес по серийному номеру перед запуском следующей фазы
- **Авто-прошивка при подключении** — автоматический старт прошивки при подключении устройства
- **Перезагрузка после прошивки** — отправка `FB: reboot` по завершении
- **Повышение прав** — поддержка `sudo` / `pkexec` для Linux; `sudo` настраивается автоматически на macOS
- **Лог-файлы** — опциональная запись `.log`-файла с временной меткой для каждого устройства в UTF-8
- **Двуязычный интерфейс** — русский и английский, переключение без перезапуска
- **Тёмная тема** — автоматически на Windows (следует системной теме)

## Типы пресетов

| Тип | Описание |
|-----|----------|
| **SimpleBin** | Одна фаза: `uuu <file.bin>` — загрузка или простая SDP-прошивка |
| **EmmcAll** | Одна фаза: `uuu -b emmc_all <bootloader> <image.wic>` — полная прошивка eMMC |
| **EmmcAll4G** | Две фазы: `uuu <4g-init.bin>`, затем `uuu -b emmc_all <bootloader> <image.wic>` — для плат с 4 ГБ LPDDR4, требующих специального бинарника инициализации RAM перед основной прошивкой |

Пресеты сохраняются в настройках приложения и не теряются между сессиями.

## Требования

- **[uuu](https://github.com/nxp-imx/mfgtools/releases)** — бинарник нужно скачать отдельно и указать путь в настройках
- **Qt 6.4+**
- **libusb 1.0** (опционально — используется как резервный способ определения устройств, если путь к uuu не задан)

## Сборка

### Linux

```bash
sudo apt install qt6-base-dev libqt6svg6-dev libusb-1.0-0-dev cmake build-essential

cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build

./build/uuuapp
```

Для работы с USB без sudo добавьте udev-правило для устройств NXP:

```bash
echo 'SUBSYSTEM=="usb", ATTRS{idVendor}=="1fc9", MODE="0666"' | sudo tee /etc/udev/rules.d/70-nxp.rules
sudo udevadm control --reload-rules
```

Без этого правила приложение автоматически запросит `pkexec` или `sudo` при обнаружении ошибки прав доступа.

### macOS

```bash
brew install qt libusb

cmake -B build -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_PREFIX_PATH="$(brew --prefix qt);$(brew --prefix libusb)"
cmake --build build

open build/uuuapp.app
```

При первом запуске macOS может заблокировать приложение, так как оно не подписано. Чтобы обойти это:

```bash
xattr -cr /path/to/uuuapp.app
```

Или: правой кнопкой → Открыть → Открыть в диалоге.

На macOS для захвата USB-устройств NXP через libusb требуется sudo — приложение настраивает это автоматически при первом запуске.

### Windows

Зависимости через [vcpkg](https://github.com/microsoft/vcpkg):

```powershell
vcpkg install libusb --triplet x64-windows
```

Установите [Qt 6](https://www.qt.io/download-qt-installer), затем:

```powershell
cmake -B build -DCMAKE_BUILD_TYPE=Release `
  -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build --config Release
windeployqt build/Release/uuuapp.exe
```

## Первоначальная настройка

После запуска откройте **Settings**:

- **UUU Binary** — путь до бинарника `uuu` ([скачать с releases](https://github.com/nxp-imx/mfgtools/releases)). Приложение автоматически ищет uuu в `PATH` и стандартных местах установки.
- **Privilege** — оставьте пустым при наличии udev-правила (Linux) или на Windows. Установите `sudo` на macOS (настраивается автоматически при первом запуске) или на Linux без udev. Используйте `pkexec` для графического запроса пароля на Linux.
- **Language** — English / Русский
- **Flash Logs** — включите для сохранения `.log`-файла с временной меткой для каждой сессии прошивки в выбранную папку.
